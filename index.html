Spreadsheet Website

Background & Usage

This is an experiment of mine, inspired by the article Use a Google Spreadsheet as your JSON backend.

If you do not see a button that says "Start application," then you are on an incompatible browser and/or have js disabled, and this will not work for you. Please try a compatible browser (or one with js enabled).

When you click "Start application," you will be asked for two things:

  • Do you want to autostart the application on your next visit?
  • What is your Google Spreadsheet backend key?

Both of these choices will be saved to the localStorage of the browser on your device, and will not be asked again.

Format of Google Spreadsheet

The google spreadsheet must be publically accessible, as per the instructions in the article referenced in the previous section.

The format of the spreadsheet is the following:

local_name parent type id classes innerHTML
container div container these are class names This is the contents of an element
another_el container div This is a child element of the previous one

Ok, so a quick lowdown:

  • SpreadSHeet ELement
  • Will make an ajax call to a google spreadsheet
  • The sheet will contain json objects for a Cel
  • A Cel is created using the Cel.js library
  • The Cel.js library enables strictly defining each element in a page, from its attributes, to its style, to its behaviour, all together, all while still enforcing modularity
